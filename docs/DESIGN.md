# 디자인 (DESIGN)

> 이 문서는 디자인 **방향과 원칙**을 담는다. 구체적 UI(색 토큰·컴포넌트·화면)는 Phase 1에서
> 실제 화면을 만들며 채운다.

## 1. 디자인 목표

- **원팀 몰입** — 앱에 들어오면 "여긴 LG 트윈스 팬의 공간"이라는 인상을 줘야 한다. 팀 정체성이
  색·로고·톤 전반에 흐른다. (중립 포털인 네이버와의 감정적 차별점)
- **데이터 가독성 우선** — 핵심은 정보(뉴스·이동·스탯 그래프)다. 장식보다 여백·대비·읽기 편함을 앞세운다.
- **모바일 우선** — 팬은 대부분 폰으로 본다. 좁은 화면·엄지 조작 기준으로 모든 결정을 내린다.

## 2. 브랜드 방향 (LG 트윈스)

- LG 트윈스의 상징색(레드 계열)을 포인트로, 화이트 베이스에 절제해 사용한다. 스탯·그래프의
  가독성을 해치지 않는 선에서 팀 색을 강조에만 쓴다.
- 팀 로고/워드마크는 헤더·아이콘에 일관되게. (실제 색 토큰과 로고 사용 규칙은 Phase 1에서 확정)
- 저작권: 팀 로고·엠블럼 사용은 개인/학습용 범위에서 신중히. 필요 시 단순화한 자체 심볼로 대체.

## 3. 시그니처 — "주식 차트" 스탯 시각화

이 앱의 정체성은 선수 스탯을 **금융 차트처럼** 보여주는 것이다.

- 타율/OPS를 시간 축 라인(또는 영역) 차트로. **일별/주별/월별 토글**로 확대·축소.
- 상승 구간과 하락(슬럼프) 구간을 색·주석으로 읽히게 한다.
- 기준선(리그 평균·시즌 초 대비 등)을 얹어 "지금 잘하는가"를 한눈에.
- 라이브러리 후보: Recharts / Chart.js (Phase 1에서 결정).

## 4. 정보 구조 (초안)

- **홈** — 우리 팀 최신 뉴스 + 주목 선수 시계열 요약.
- **뉴스** — LG 뉴스 목록(헤드라인·날짜 → 원문 링크).
- **선수** — 선수 선택 → 스탯 시계열 그래프(핵심 화면).
- **이동** — 등록/말소·콜업·트레이드 리스트 + 사유. (Phase 2)

## 5. 향후 (Phase 1에서 채움)

- 색 팔레트·타이포·컴포넌트 토큰 확정.
- 실제 화면 시안과 스크린샷.
- 그래프 인터랙션(호버·기간 선택) 상세.
