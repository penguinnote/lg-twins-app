# 선수 능력치 레이더 (STATUS_RATINGS)

> 선수 상태창의 육각 레이더. 6능력치, 각 0~100 = **리그 백분위**. 코인 캔들이 "시간에 따른 폼
> 흐름"이라면, 레이더는 **"이 선수는 어떤 유형인가"라는 능력 프로필 스냅샷**이다.
> 전부 KBO 박스스코어로 계산한다. 수비·구속·회전수·멘탈·작전은 공개 데이터가 없어 제외(코인
> 모델과 동일한 한계).

## 타자 6축

| 축 | 기반 지표 | 방향 |
|---|---|---|
| 컨택 | 타율 AVG (= H/AB) | 높을수록 ↑ |
| 파워 | ISO (= SLG − AVG) | ↑ |
| 선구안 | 볼넷율 BB% (= BB / PA) | ↑ |
| 주력 | 도루 SB (+ 3루타 가산) | ↑ |
| 삼진회피 | 삼진율 K% (= SO / PA) | **낮을수록 좋음(역방향)** |
| 생산성 | wOBA (코인 모델의 wRAA 계산 재사용) | ↑ |

## 투수 6축

| 축 | 기반 지표 | 방향 |
|---|---|---|
| 제구 | BB/9 | **역방향** |
| 탈삼진 | K/9 | ↑ |
| 피안타억제 | 피안타율(피안타/상대타수) 또는 H/9 | **역방향** |
| 장타억제 | 피홈런/9 (HR/9) | **역방향** |
| 이닝소화 | 총 이닝 IP | ↑ (선발>구원 실제 차이 반영) |
| 실점억제 | FIP (또는 ERA) | **역방향** |

## 정규화 — 리그 백분위(0~100)

- **각 지표에서 리그 전체(자격 충족) 선수 중 백분위**를 능력치로 삼는다. "리그 상위 X%"라 직관적.
- 계산: `rating = (그 지표에서 나보다 낮은(또는 역방향이면 높은) 선수 수) / 자격 선수 수 × 100`.
  역방향 지표(삼진율·BB/9·ERA 등)는 방향을 반전해 "좋을수록 높은 점수"가 되게 한다.
- **자격 기준**: 타자 최소 PA(예: 50), 투수 최소 IP(예: 10). 미달 선수는 표시하되 **"표본 부족"**
  배지를 달고 능력치는 회색/흐리게(과신 방지).
- **OVR(종합)** = 6축 평균(원하면 가중평균).

## 데이터 근거 & 신규 수집

- 백분위를 내려면 **리그 전체 분포**가 필요하다 → KBO 시즌 기록 리더보드(타자·투수 전체 자격
  선수)를 **신규 스크랩**한다. (LG 선수 개별 지표는 기존 경기별 로그 합산으로 이미 확보.)
- 리그 분포는 매일 조금씩 바뀌므로 일일 파이프라인에서 갱신.

## 한계 (사이트에 명시)

- 시즌 초반은 표본이 작아 백분위가 출렁인다 → 자격 기준 + "표본 부족" 표기로 완화.
- **수비·구속·회전수·멘탈·작전 실행은 데이터가 없어 제외.** 즉 "박스스코어 기반 능력 근사"이며
  완전한 선수 능력이 아님을 고지(코인 모델과 동일한 투명성).
