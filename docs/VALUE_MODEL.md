# 코인 가격 모델 (VALUE_MODEL)

> "트윈스 코인" — 선수의 가치 기여를 **코인 시세**처럼 나타내는 지수.
> 가격이 오르면 떡상, 내리면 떡락. KBO 경기별 박스스코어로 **자체 계산**한다(스파이크2로 확인).

## 1. 개념

- **코인 가격 = 누적 가치.** 각 경기 = 그날의 등락(가치 기여). `open = 경기 전 가격`, `close = 경기 후 가격`.
- 잘한 경기 → 가격 상승(떡상), 못한 경기 → 하락(떡락). 연속 상승 = "코인 떡상 중".
- WPA·클러치·수비는 KBO 공개 데이터로 불가(스파이크2 확정) → **제외**. 박스스코어 기반 가치로 근사.

## 2. 타자 그날 가치 — 타격 런(wRAA)

선형가중치(wOBA) 기반 "평균 대비 타격 런".

```
1B      = H − 2B − 3B − HR
wOBA_g  = (w_BB·BB + w_HBP·HBP + w_1B·1B + w_2B·2B + w_3B·3B + w_HR·HR) / (AB + BB + HBP)
wRAA_g  = ((wOBA_g − lgwOBA) / wOBA_scale) × PA        (PA ≈ AB + BB + HBP)
주루보정 = +0.20·SB − 0.42·CS
타자_가치 = wRAA_g + 주루보정
```

- 가중치(대표값, **KBO 2026 상수로 보정 예정**): `BB .69, HBP .72, 1B .89, 2B 1.27, 3B 1.62, HR 2.10`
- SF·IBB 없음 → 분모에서 SF 생략(근사). BB는 uBB로 취급.
- 결과: 평균 경기 ≈ 0, 홈런·장타·출루 많을수록 크게 +.

## 3. 투수 그날 가치 — 막아낸 실점 + 역할 보너스

```
RP_g        = (lgRA9 / 9) × IP − R          (평균 투수가 그 이닝에 내줄 실점 − 실제 실점)
역할보너스   = b_HLD·HLD + b_SV·SV           (휴리스틱: 홀드 +0.3, 세이브 +0.5 run)
투수_가치    = RP_g + 역할보너스
```

- IP 분수표기(`5 2/3`) → 5.667로 파싱.
- `R`(실점, 결과 중심) 사용. FIP·ER는 "과정" 보조 뷰로 나중에 병행 가능.
- `lgRA9` = 리그 9이닝당 실점(자체 산출). 역할 보너스는 레버리지 데이터가 없어 붙인 근사이며, **사이트에 "휴리스틱"으로 명시**.

## 4. 코인 가격(스케일)

- 타자·투수 모두 **런(runs) 단위** → 공통 스케일에 놓여 포지션 넘어 비교 가능.
- `coin_price_i = coin_price_{i-1} + value_game_i × SCALE`
- `base = 100`, `SCALE = 튜닝값(기본 5)` → "코인 시세" 느낌. (스케일은 순전히 표현용, 곡선 모양이 본질)
- 결장/미등판(타석·이닝 0) → 가치 0 → 가격 유지, **캔들 없음(짝대기)**.
- 참고: 타자는 매일 소폭, 선발은 5일마다 큰 폭으로 움직여 코인 텍스처가 다름(정상). 필요 시 역할별
  변동성 정규화 검토.

## 5. 캔들 의미 (앞선 디자인과 통합)

- `open` = 경기 전 코인가, `close` = 경기 후 코인가.
- `close > open` → 상승(빨강, 떡상) / `close < open` → 하락(파랑, 떡락).
- **기준선(수면선) = "평균 경기(가치 0)".** ← 이전의 고정 .270을 대체. "평균 대비"라 더 원칙적.
- **캔들 크기 = |close − open| = 그날 가치 크기.** 홈런 경기 = 큰 상승 캔들, 무안타 = 하락 캔들
  (자연히 타수 반영 = 선택 B안).
- 이동평균선 = 가치 추세(진짜 신호). 손가락 패닝·핀치 줌.

## 6. 리그 상수 & 보정

`w_*, lgwOBA, wOBA_scale, lgRA9` 는 연도·리그 의존값이다. Statiz가 robots로 막혀 있으니:
- **자체 산출**: 파이프라인이 수집한 리그 전체 합계에서 KBO 2026 상수를 계산(권장), 또는
- **수동 입력**: 공개된 KBO 연도 상수를 상수 파일로.
- v1은 대표값으로 시작 → 리그 집계가 쌓이면 보정.

## 7. 한계 (반드시 사이트에 고지)

- **수비·주루(SB/CS 외)·클러치/레버리지(WPA)·포지션·대체선수 보정 없음.**
- 즉 **"박스스코어 기반 타격/투구 기여 지수"이지 WAR가 아니다.** "완전한 선수 가치가 아님"을 명시.
- 단일 경기는 표본이 작아 노이즈 → **누적 가격 + 이동평균**으로 해석(단일 캔들 과신 금지).
- 이 투명성 자체가 분석 신뢰도를 준다.

## 8. 데이터 근거 (스파이크2 확정)

- **타자** ✅ 1B(유도)·2B·3B·HR·BB·HBP·SB·CS·RBI·R·SO·GDP  ✗ SF·SH·IBB
- **투수** ✅ IP·H·HR·BB·HBP·SO·R·ER·승패·세이브·홀드·선발/구원·TBF  ✗ 투구수(TBF 대용)
- ✗ WPA·경기별 수비·Statiz(robots 전면 차단) → 공식에서 제외
