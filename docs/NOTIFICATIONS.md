# 알림(푸시) 설계 (NOTIFICATIONS)

> 저녁 경기 종료 후 코인 시세가 갱신되면 푸시 알림. 두 종류:
> (1) **전체 방송** — 오늘 최대 등락 선수, (2) **개인화** — 내가 팔로우한 선수의 등락.
> Convene의 FCM 푸시 시스템을 재활용한다.

## 1. 계정 없이 — 익명 인증

- Firebase Auth **익명 로그인**으로 브라우저별 `uid` 부여(로그인 없음). Convene 방식 그대로.
- 팔로우·알림 토큰을 `uid`에 묶어 저장. v1은 **기기 단위**(로그인 붙이면 기기 간 동기화 가능).

## 2. 데이터 모델 (Firestore)

```
subscribers/{uid} {
  token: FCM 토큰,
  followedPlayers: [playerId, ...],
  platform: 'ios'|'android'|'other',
  updatedAt
}
```

## 3. 팔로우 UI (프론트)

- 선수 카드/상세에 **팔로우 토글(별·알림 아이콘)**. 켜면 `subscribers/{uid}.followedPlayers`에 추가.
- 팔로우/알림을 **처음 켤 때** 알림 권한 요청 + FCM 토큰 등록(Convene의 enablePush 재활용).

## 4. 발송 로직 (저녁 파이프라인)

코인 갱신 후, 각 선수의 그날 코인 등락 Δ(= 오늘 close − 어제 close)를 계산한다.

- **(1) 전체 방송** — |Δ|가 가장 큰 선수 1명 → **모든 subscriber**에게:
  "오늘 최대 등락 · 오스틴 코인 +14 떡상 🔴" (Δ>0 떡상 / Δ<0 떡락).
- **(2) 개인화** — 각 subscriber의 `followedPlayers` 중 **그날 경기해서 코인이 변한** 선수 →
  그 사람 토큰으로: "팔로우한 문보경 −8 떡락" (여러 명이면 요약: "오스틴 +12, 문보경 −8").
- **중복 방지**: 최대 등락 선수를 팔로우한 사람은 방송/개인화 중 하나만 받게.
- **발송 주체**: GitHub Actions 파이프라인이 **Firebase Admin**(서비스계정 키 = GitHub 시크릿)으로
  `subscribers`를 읽고 FCM 발송. **data-only 페이로드**(Convene 교훈 — notification 필드 넣으면 중복).

## 5. 타이밍

- 저녁 경기 종료 후(예: **밤 11:30 KST**) 코인 파이프라인을 실행 → 새 경기로 코인이 갱신되면 발송.
  (없으면 발송 스킵.)

## 6. 서비스워커 · 설치

- 웹앱에 SW 추가(푸시 수신 + 알림 클릭 시 해당 선수 화면으로 이동). 사이트가 **설치형 PWA**가 된다
  (설치는 알림 원하는 사람만 선택).

## 7. iOS 주의 (Convene과 동일)

- iOS는 **홈 화면에 추가(설치) 후에만** 웹 푸시가 된다 → "알림 받으려면 홈 화면에 추가" 안내 UI.
- 안드로이드/데스크톱은 브라우저에서 바로 가능.

## 8. 한계 · 주의

- 팔로우는 기기 단위(로그인 없음, v1).
- data-only 유지, 알림마다 고유 tag로 덮어쓰기 방지(Convene 교훈).
- 발송량 소규모라 비용/부하 무리 없음. Firestore·FCM 무료 티어 내.
