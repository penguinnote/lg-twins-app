# 기획 (PLANNING)

> 트윈스 코인 — LG 트윈스 선수의 가치를 "코인 시세"처럼 보여주는 웹사이트.

## 1. 문제

KBO 팬은 선수의 성적을 여러 곳에 흩어진 *현재 값*으로만 소비한다. 네이버 스포츠는 종합·중립
백과사전이라 "현재 타율 .298"은 보여줘도, **선수가 팀에 만들어내는 가치가 시간에 따라 어떻게
오르내리는지**는 못 보여준다.

정작 팬들은 이미 그걸 **"창기 코인 떡상", "OO 코인 떡락"**처럼 시세로 느끼고 말한다. 그런데
그 감각을 실제로 구현한 도구는 없다.

## 2. 컨셉과 차별점

**트윈스 코인 = 선수의 경기별 기여(가치)를 누적한 지수를 "코인 시세"로 시각화.** 네이버가
구조적으로 안 하는 세 축에서 차별화한다.

- **원팀 몰입** — LG 트윈스 팬 전용. 우리 팀 선수만, 우리 팀 색으로.
- **가치 지수(퀀트)** — 타율 같은 단순 rate가 아니라, 홈런·장타·출루·호투를 반영한 **가치**를
  직접 계산. 팬의 "코인 떡상" 밈을 데이터로.
- **코인/캔들 경험** — 주식 시세처럼 캔들·이동평균·확대. 잘하면 떡상, 못하면 떡락이 눈에 보인다.

## 3. 타깃 유저

- 1차: **LG 트윈스 팬.** 개인 지인이 아니라 온라인 LG/KBO 커뮤니티가 실제 유저층.
- 각 팬이 자기 최애 선수의 "코인"을 지켜보는, 개별 가치가 명확한 제품.

## 4. 핵심 기능

| 기능 | 설명 |
|---|---|
| 트윈스 코인 시세판(홈) | 타자·투수 코인 시세 랭킹 + 미니 시세선 (타/투 배지) |
| **선수 코인 (캔들 차트)** | 경기별 캔들(떡상/떡락) + 이동평균선, 코인/타율·OPS 토글, 일·주·월 |
| 우리 팀 뉴스 피드 | LG 뉴스 헤드라인·날짜, 탭하면 원문 |
| (로드맵) 로스터 이동 트래커 | 등록/말소·트레이드 + LLM 사유 |

## 5. 코인 가격은 어떻게 정하나 (요약)

경기별 박스스코어로 **가치를 직접 계산**한다. 상세: [VALUE_MODEL.md](VALUE_MODEL.md).

- **타자** = wOBA 기반 타격 런(wRAA) + 주루.
- **투수** = 평균 대비 막아낸 실점 + 세이브·홀드 역할 보너스.
- 둘 다 런(runs) 단위 → **코인 가격 = 누적 가치 × 스케일 + 100.** 캔들 = 그날 등락.

**정직한 한계(명시):** 수비·클러치(WPA)·대체선수 보정은 KBO 공개 데이터로 불가 → 제외.
"박스스코어 기반 기여 지수이지 WAR가 아님"을 사이트에 상시 고지. 단일 경기는 노이즈 →
누적 가격·이동평균으로 해석.

## 6. 데이터 실현가능성을 먼저 검증했다 (스파이크)

KBO는 공개 API가 없어, 본격 개발 전에 데이터가 실제로 긁히는지 시간을 정해 검증했다.

- **스파이크 1** — 팀 뉴스 ✅ / 선수 경기별 로그 ✅(과거 소급·공식값 15/15 일치) / 등록말소 ⚠️(당일만).
- **스파이크 2** — 투수 경기별 ✅ / 타자 공격 필드 ✅ / 경기별 WPA·수비 🔴 / Statiz 🔴(robots 차단).
  → 결론: **코인 가격은 박스스코어로 자체 계산**, 수비·클러치는 제외.

"만들기 전에 데이터부터 검증"이 이 프로젝트의 방식이다.

## 7. 핵심 의사결정과 근거

- **웹사이트(설치형 아님)** — 무작위 인터넷 유저 대상이라 링크로 여는 웹이 마찰 최저.
- **LG 한 팀** — 원팀 몰입이 곧 차별점. 범위를 좁혀 완성도.
- **DB 없이 JSON** — 읽기 전용·일 배치라 DB가 과함. 파이프라인이 JSON 생성, 웹이 읽음.
- **타율 트래커 → 코인(가치 지수)로 피벗** — 타율 시계열은 후반부 평평해 정보가 약했고, 팬이
  실제로 쓰는 "코인" 은유가 더 강한 제품이자 퀀트 차별점이 됨.
- **WPA 대신 박스스코어 자체 계산** — WPA·Statiz가 데이터/규약상 불가라, 얻을 수 있는 데이터로
  방어 가능한 가치식을 직접 설계.

## 8. MVP & 로드맵

- **Phase 0(완료)** — 데이터 파이프라인 + 매일 자동 수집.
- **Phase 1(완료)** — 뉴스 + 선수 시계열 웹사이트 배포.
- **트윈스 코인(완료)** — 경기별 가치 지수 + 코인 가격 + 캔들 차트.
- **다음** — 가치 모델 보정(리그 상수·역할 보너스), 로스터 이동 + LLM 사유, (선택) 승부 예측·알림.

## 9. 성공 지표

- **핵심: "팬이 네이버 대신 이 코인을 열어보는가"** → 재방문·리텐션.
- 어떤 선수 코인을 많이 보나, 시세판 사용률.
- (커뮤니티 배포 후) 사용 로그 기반 검증.
