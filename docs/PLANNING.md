# 기획 (PLANNING)

> LG 트윈스 팬을 위한 웹사이트 — 우리 팀 뉴스·로스터 이동·선수 스탯 시계열을 한 곳에서.

## 1. 문제

KBO 팬은 정보를 여러 곳에 흩어진 채로 소비한다. 특히 네이버 스포츠는 **종합·중립 백과사전**이라
한 팀 팬 입장에선 세 가지가 불편하다.

1. **우리 팀 뉴스만 보기 귀찮다** — 모든 팀 기사가 섞여 있어 내 팀 소식을 골라내야 한다.
2. **로스터 이동을 한눈에 못 본다** — 누가 2군에 내려갔고 누가 콜업·트레이드됐는지 흩어져 있다.
3. **선수의 '흐름'을 볼 수 없다** — "현재 타율 .298"만 보여줄 뿐, 그 선수가 어떻게 성장하고
   슬럼프에 빠졌는지의 **시간 축이 없다.**

## 2. 차별점 — 네이버가 '못' 하는 것

네이버의 강점(포괄성)과는 경쟁하지 않는다. 네이버가 구조적으로 안 하는 축에서 이긴다.

- **원팀 몰입** — LG 트윈스 팬 전용. 우리 팀 정보만, 우리 팀 색으로. 중립 포털이 줄 수 없는 경험.
- **시계열(핵심)** — 타율/OPS를 **주식 차트처럼** 시간 축으로. "6월 전성기 → 7월 슬럼프"를
  눈으로 보여준다. 어떤 대형 포털도 이렇게 제공하지 않는다.
- **큐레이션** — 로스터 이동에 사유를 붙이고, 우리 팀 관점으로 정리한다.

원팀 몰입은 **그릇**, 시계열·큐레이션은 **알맹이**다. 둘 다 있어야 "네이버 대신 이걸 연다"가 성립한다.

## 3. 타깃 유저

- 1차: **LG 트윈스 팬.** MVP는 내 주변 KBO 보는 친구들로 검증한다.
- 개별 유저에게 독립적으로 가치를 준다(네트워크 효과 불필요) → 소규모로도 검증 가능.

## 4. 핵심 기능

| 기능 | 설명 |
|---|---|
| 우리 팀 뉴스 피드 | LG 뉴스 헤드라인·날짜, 탭하면 원문으로(본문 미저장) |
| 로스터 이동 트래커 | 등록/말소·콜업·트레이드 리스트 + 사유 한 줄(뉴스+LLM 생성) |
| **선수 스탯 시계열 그래프** | 타율/OPS를 주식 차트처럼. 일별/주별/월별 토글 (핵심 차별점) |
| (후반·선택) 승부 예측 | 다음 경기 승리 확률 + 근거. 시계열 피처 재활용 |

## 5. 데이터 실현가능성을 먼저 검증했다 (데이터 스파이크)

이 프로젝트의 성패는 데이터에 달려 있고, KBO는 공개 API가 없다. 그래서 **본격 개발 전에**
세 데이터 소스가 실제로 수집되는지 시간을 정해 검증했다(throwaway 스파이크).

| 소스 | 결과 |
|---|---|
| 팀 뉴스 | ✅ 헤드라인·링크·날짜 확보 |
| 로스터 이동 | ⚠️ 당일만 제공·소급 불가 → **매일 폴링해 누적**하도록 설계 |
| 선수 시계열 | ✅ 경기별 로그로 **과거까지 완전 복원.** 누적 타율이 공식값과 **15/15 정확 일치** |

결론: 원안대로 가능. 다만 로스터 이동은 소급이 안 되므로 **수집 파이프라인을 가장 먼저 가동**했다.

## 6. 핵심 의사결정과 근거

- **웹사이트(설치형 앱/PWA 아님)** — 타깃이 무작위 인터넷 유저라 설치 안내가 불가능하다.
  링크만 누르면 열리는 웹사이트가 마찰이 가장 낮고, 앱스토어 비용도 없다. (푸시가 필요해지면
  나중에 '홈 화면 추가'를 선택 기능으로 얹는다.)
- **LG 한 팀 집중** — 10팀 확장 대신 한 팀에 깊이. 원팀 몰입이 곧 차별점이라 범위를 좁혀 완성도를 높인다.
- **DB 없이 JSON 저장** — MVP는 읽기 전용·일 배치라 DB가 과하다. 파이프라인이 JSON을 만들고
  웹사이트가 읽는다. (계정·알림이 필요해지면 Firestore로 승격.)
- **GitHub Actions 자동화** — 서버 없이 매일 수집. 파이프라인이 저장소에 드러나 재현·검증이 쉽다.

## 7. MVP & 로드맵

- **Phase 0 (완료)** — 데이터 파이프라인 + 매일 자동 수집. 로스터 이동 이력 축적 시작.
- **Phase 1 (MVP)** — 뉴스 + 시계열 그래프 웹사이트. 친구들에게 배포해 검증.
- **Phase 2** — 로스터 이동 탭 + LLM 사유 생성.
- **Phase 3** — (선택) 승부 예측 모델, 알림(선택적 설치).

## 8. 성공 지표

- **핵심: "친구가 네이버 대신 이 앱을 여는가"** → 재방문·리텐션.
- 시계열 탭 사용률, 뉴스 클릭률.
- (Phase 3) 예측 정확도 vs 베이스라인.
